# 静的解析設定

## 概要

本プロジェクトでは Tailwind CSS v4 の独自ディレクティブ（`@config`、`@theme`、`@apply` など）を使用しているため、静的解析ツールが誤検知エラーを出す場合がある。この問題を解消するために、Biome と VS Code の両方に設定を追加している。

---

## 問題の背景

Tailwind CSS v4 では、CSS ファイル内で以下のような独自のディレクティブを使用する。

```css
@config "../tailwind.config.ts";
@custom-variant dark (&:is(.dark *));
@theme inline { ... }
@apply bg-background text-foreground;
```

これらは標準の CSS 仕様には存在しないため、2種類のツールが誤検知を発生させる。

| ツール | レベル | エラー内容 |
|--------|--------|-----------|
| **Biome**（CSS パーサー） | ERROR | `Tailwind-specific syntax is disabled` |
| **VS Code CSS Language Server** | WARNING | `Unknown at rule @config` 等 |

いずれも**実際のバグではなく、ツール側が Tailwind 構文を認識できないことが原因**である。ビルド（`vite build`）は正常に成功する。

---

## 解決策

### 1. Biome — `tailwindDirectives` の有効化

`biome.jsonc` に `css.parser.tailwindDirectives` を追加し、Tailwind 固有のディレクティブをパーサーに認識させる。

```jsonc
{
  "css": {
    "parser": {
      "tailwindDirectives": true
    }
  }
}
```

この設定により、Biome が `@config`、`@theme`、`@apply`、`@custom-variant` などを正しくパースできるようになり、ERROR が解消される。

#### 補足

- この設定は Biome v2.3 以降で利用可能。
- 本プロジェクトでは Biome v2.4.7 を使用している。

### 2. VS Code — `unknownAtRules` の無効化

`.vscode/settings.json` を作成し、VS Code 組み込みの CSS Language Server による `Unknown at rule` 警告を抑制する。

```json
{
  "css.lint.unknownAtRules": "ignore"
}
```

この設定により、`@config`、`@theme`、`@apply` 等に対する WARNING が表示されなくなる。

#### 補足

- `.vscode/settings.json` はプロジェクトローカルの設定であり、リポジトリに含めることでクローンした全メンバーに適用される。
- ユーザーのグローバル設定（`~/Library/Application Support/Code/User/settings.json`）には影響しない。
- Cursor を使用している場合も、この設定は同様に機能する。

---

## ファイル構成

```
├── .vscode/
│   └── settings.json       # VS Code / Cursor のプロジェクトローカル設定
├── biome.jsonc              # Biome の設定（リンター・フォーマッター）
```

---

## 設定の効果まとめ

| 対象ツール | 設定ファイル | 設定項目 | 効果 |
|-----------|------------|---------|------|
| Biome | `biome.jsonc` | `css.parser.tailwindDirectives: true` | Tailwind ディレクティブの ERROR を解消 |
| VS Code CSS Language Server | `.vscode/settings.json` | `css.lint.unknownAtRules: "ignore"` | `Unknown at rule` の WARNING を解消 |

---

## 注意事項

- `css.lint.unknownAtRules` を `"ignore"` にすると、Tailwind 以外の本当に不正な at-rule も警告されなくなる。ただし、Biome 側で CSS の構文チェックは行われるため、実用上の問題はない。
- 将来的に VS Code の CSS Language Server が Tailwind v4 のディレクティブに対応した場合、`.vscode/settings.json` の設定は不要になる可能性がある。

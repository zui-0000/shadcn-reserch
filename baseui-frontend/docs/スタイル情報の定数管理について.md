# スタイル情報の定数管理について

## 概要

Tailwind CSS のプロジェクトでは、z-index やカラーなどのスタイル値を TypeScript の定数として一元管理し、Tailwind のユーティリティクラスと TypeScript コードの両方から参照できるようにする仕組みを導入している。

---

## 背景と課題

スタイルに関する値（z-index、プロジェクト固有のカラーなど）を管理する際、以下の課題がある。

- **CSS と TypeScript で値が二重管理になりがち** — CSS 変数で定義した値を TS 側でも使いたい場合、同じ値を2箇所に書くことになる
- **変更時の追従漏れ** — 片方だけ更新して不整合が起きるリスクがある

これを解決するため、**TypeScript の定数ファイルを唯一の情報源（Single Source of Truth）** とし、Tailwind の設定ファイル経由でクラス名としても利用可能にしている。

---

## アプローチの選択肢

値の管理方法として、大きく2つのアプローチがある。

### 方法1: CSS カスタムプロパティを源泉にする

CSS 変数で値を定義し、`@theme` で Tailwind に登録。TS 側からは `getComputedStyle` で参照する。

- **メリット** — Tailwind v4 の CSS ファーストの設計思想に最も合致する
- **デメリット** — TS 側からの参照がランタイム依存（`getComputedStyle`）になり、型安全性がない
- **適したケース** — ダークモードで値が変わるカラー（shadcn のテーマカラーなど）

### 方法2: TypeScript の定数を源泉にする（本プロジェクトで採用）

TS ファイルで値を定義し、`tailwind.config.ts` + `@config` ディレクティブ経由で Tailwind に登録する。

- **メリット** — 型安全、1箇所で管理、TS のどこからでもインポート可能
- **デメリット** — `tailwind.config.ts` が必要になる（v4 では本来不要）
- **適したケース** — z-index、プロジェクト固有の固定カラーなど、テーマで変動しない値

### 本プロジェクトでの方針

| 値の種類 | 管理方法 | 理由 |
|---------|---------|------|
| shadcn テーマカラー（primary, background 等） | CSS カスタムプロパティ（従来通り） | ダークモードで値が切り替わるため |
| z-index | TypeScript 定数 | テーマで変動せず、TS ロジック内でも参照するため |
| プロジェクト固有のカラー | TypeScript 定数 | 固定値であり、TS から直接参照したいため |

## ファイル構成

```
├── tailwind.config.ts          # TS 定数を Tailwind に登録
├── src/
│   ├── index.css               # @config ディレクティブで config を読み込み
│   └── styles/
│       ├── z-index.ts          # z-index の定数定義
│       └── colors.ts           # プロジェクト固有カラーの定数定義
```

## 各ファイルの役割

### src/styles/z-index.ts

z-index の値を定数として定義する。

```typescript
export const zIndex = {
  dropdown: "100",
  sticky: "200",
  overlay: "300",
  modal: "400",
  toast: "500",
  tooltip: "600",
} as const;
```

### src/styles/colors.ts

プロジェクト固有のカラーを定数として定義する。

```typescript
export const CUSTOM_COLORS = {
  "brand-blue": "#3b82f6",
  "brand-red": "#ef4444",
} as const;
```

### tailwind.config.ts

TS 定数をインポートし、Tailwind のテーマを拡張する。

```typescript
import type { Config } from "tailwindcss";
import { zIndex } from "./src/styles/z-index";
import { CUSTOM_COLORS } from "./src/styles/colors";

export default {
  theme: {
    extend: {
      zIndex: zIndex,
      colors: CUSTOM_COLORS,
    },
  },
} satisfies Config;
```

### src/index.css

`@config` ディレクティブで `tailwind.config.ts` を読み込む。

```css
@import "tailwindcss";
@import "tw-animate-css";
@import "shadcn/tailwind.css";
@import "@fontsource-variable/geist";
@config "../tailwind.config.ts";
```

## 使い方

### Tailwind のユーティリティクラスとして使う

```tsx
{/* z-index */}
<div className="z-modal">...</div>
<div className="z-tooltip">...</div>

{/* colors */}
<div className="bg-brand-blue text-brand-red">...</div>
```

### TypeScript から直接参照する

```tsx
import { zIndex } from "@/styles/z-index";
import { CUSTOM_COLORS } from "@/styles/colors";

// style prop で使用
<div style={{ zIndex: zIndex.modal }}>...</div>
<div style={{ color: CUSTOM_COLORS["brand-blue"] }}>...</div>

// ロジック内で使用
const isAboveModal = currentZIndex > Number(zIndex.modal);
```

## 定数のキー命名規則

定数のキー名は**そのまま Tailwind のクラス名の一部になる**ため、命名に注意が必要。

| キーの命名 | Tailwind クラス例 | TS 参照 | 備考 |
|-----------|------------------|---------|------|
| `brand-blue`（ケバブケース） | `bg-brand-blue` | `["brand-blue"]` | Tailwind 規約に最も自然 |
| `brandBlue`（キャメルケース） | `bg-brandBlue` | `.brandBlue` | TS のドット記法で書ける |
| `BRAND_BLUE`（大文字スネーク） | `bg-BRAND_BLUE` | `.BRAND_BLUE` | 動作するがクラス名として不自然 |

### 推奨

- **Tailwind クラスで使う値** → ケバブケース（`brand-blue`）またはキャメルケース（`brandBlue`）
- **TS からのみ参照する値** → 命名規則は自由（大文字スネークケースでも可）

## 関連ツール・ライブラリ

定数の管理をさらに発展させたい場合に検討できるツール。

| ツール | 概要 | Tailwind との併用 |
|-------|------|------------------|
| **Style Dictionary**（Amazon 製） | JSON/YAML でデザイントークンを定義し、CSS・TS など複数形式に自動生成 | 可能 |
| **Panda CSS** | CSS-in-JS フレームワーク。TS でトークンを定義し、ゼロランタイムで CSS を生成 | 不可（Tailwind を置換） |
| **UnoCSS** | JS/TS ファーストの CSS エンジン。設定がすべて JS/TS で完結 | 不可（Tailwind を置換） |

本プロジェクトの規模では `tailwind.config.ts` + TS 定数の方式で十分であり、追加のライブラリは不要と判断している。

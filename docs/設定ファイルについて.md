# 設定ファイルについて

## 概要

shadcn/ui + Tailwind CSS v4 + Vite 構成では、設定ファイルが大幅に簡素化されている。以前（Tailwind v3）まで必要だった `tailwind.config.js` や `postcss.config.js` は不要になり、最小限の設定で動作する。

---

## components.json

shadcn/ui の CLI（`pnpm dlx shadcn add` など）が参照する設定ファイル。プロジェクトルートに配置する。

### 注意事項

- ファイル名は `components.json` 固定。CLI がこの名前をハードコードで探すため、`components.jsonc` などにリネームすると動作しない。
- コメント付き JSON（JSONC）にも非対応。標準の JSON パーサーで読み込まれるため、コメントを書くとパースエラーになる。

### 設定項目一覧

#### 基本設定

| 項目 | 説明 |
|---|---|
| `$schema` | JSON スキーマの URL。エディタ上で入力補完やバリデーションを有効にする。 |
| `style` | コンポーネントのスタイルプリセット。`"default"`, `"new-york"`, `"base-nova"` などがある。 |
| `rsc` | React Server Components を使用するかどうか。`true` にすると `"use client"` ディレクティブが自動付与される。Vite ベースのプロジェクトでは `false`。 |
| `tsx` | コンポーネントを `.tsx`（TypeScript）で生成するかどうか。`false` だと `.jsx` で生成される。 |

#### `tailwind` セクション

| 項目 | 説明 |
|---|---|
| `config` | Tailwind の設定ファイルのパス。v4 では CSS ベースの設定に移行したため空文字でよい。 |
| `css` | グローバル CSS ファイルのパス。shadcn が CSS 変数（カラーテーマなど）をここに書き込む。 |
| `baseColor` | ベースカラーのプリセット。`"slate"`, `"gray"`, `"zinc"`, `"neutral"`, `"stone"` から選択。灰色系の色味が変わる。 |
| `cssVariables` | CSS カスタムプロパティ（変数）でテーマを管理するかどうか。`true` が推奨で、ダークモード対応が容易になる。 |
| `prefix` | Tailwind のクラス名にプレフィックスを付ける場合に指定（例: `"tw-"` → `tw-bg-red-500`）。他の CSS フレームワークとの競合回避に使う。 |

#### その他

| 項目 | 説明 |
|---|---|
| `iconLibrary` | 使用するアイコンライブラリ。`"lucide"` が標準。 |
| `rtl` | Right-to-Left レイアウトのサポート。日本語では `false` で問題ない。 |
| `aliases` | `npx shadcn add` で生成されるコードの import パスエイリアスを制御する。`components`, `utils`, `ui`, `lib`, `hooks` の配置先を指定。 |
| `menuColor` | shadcn CLI のメニュー表示色の設定。 |
| `menuAccent` | shadcn CLI のメニューアクセント色の設定。 |
| `registries` | カスタムコンポーネントレジストリの設定。サードパーティや社内のレジストリを追加する場合に使用。 |

## Tailwind CSS の設定：v3 と v4 の違い

### v3 まで（旧方式）

以下の 2 つの設定ファイルが必要だった。

**postcss.config.js** — PostCSS のプラグインとして Tailwind を登録

```javascript
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  },
}
```

**tailwind.config.js** — Tailwind のテーマやスキャン対象を定義

```javascript
module.exports = {
  content: ["./src/**/*.{js,ts,jsx,tsx}"],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

### v4（現在の方式）

v4 ではアーキテクチャが大きく変わり、設定ファイルが不要になった。

| 変更点 | 詳細 |
|---|---|
| PostCSS 設定が不要 | Tailwind が Vite プラグインとして直接統合された |
| `tailwind.config.js` が不要 | 設定は CSS ファイル内の `@theme` ディレクティブで行う |
| `content` の指定が不要 | クラス名の使用箇所が自動検出されるようになった |

**vite.config.ts** — Vite プラグインとして登録するだけ

```typescript
import tailwindcss from "@tailwindcss/vite"

export default defineConfig({
  plugins: [
    tailwindcss(),
  ],
})
```

**src/index.css** — CSS ファイル内で設定

```css
@import "tailwindcss";

@theme {
  --color-primary: oklch(0.5 0.2 240);
  /* テーマのカスタマイズはここで行う */
}
```

### 比較まとめ

| | v3 以前 | v4 |
|---|---|---|
| `postcss.config.js` | 必要 | **不要** |
| `tailwind.config.js` | 必要 | **不要**（CSS 内で設定） |
| `content` の指定 | 必要 | **不要**（自動検出） |
| 動作方式 | PostCSS プラグイン | Vite プラグイン |

設定ファイルが大幅に減り、CSS ファイル 1 つに集約されたのが v4 の大きな特徴である。

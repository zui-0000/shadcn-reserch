# shadcnで生成されたコンポーネントの運用方針

## shadcn/ui の設計思想

一般的な UI ライブラリ（MUI, Chakra UI など）は `node_modules` にあるコンポーネントをそのまま使うが、shadcn/ui は**ソースコードをプロジェクトにコピーする**方式を取っている。`src/components/ui/` のファイルはすべて自分のコードであり、自由にカスタマイズできる。

## 使い方の 3パターン

### 1. そのまま Atom として使う

variant や size などの props で十分なら、編集せずにそのまま使う。

```tsx
import { Button } from "@/components/ui/button";

<Button variant="outline" size="sm">Click</Button>
```

### 2. 直接編集する

プロジェクト固有の要件がある場合、`ui/` のファイルを直接編集する。例えば独自のバリアントを追加するなど。

```tsx
// button.tsx の variants に追加
variant: {
  default: "...",
  outline: "...",
  brand: "bg-blue-600 text-white hover:bg-blue-700",  // 追加
},
```

### 3. Atom を組み合わせて独自コンポーネントを作る（推奨パターン）

`ui/` のコンポーネントを Atom として扱い、それを組み合わせた Molecule / Organism を別途作る。

```tsx
// components/SearchField/index.tsx
import { Input } from "@/components/ui/input";
import { Button } from "@/components/ui/button";
import { Search } from "lucide-react";

export function SearchField() {
  return (
    <div className="flex gap-2">
      <Input placeholder="検索..." />
      <Button variant="outline" size="icon">
        <Search />
      </Button>
    </div>
  );
}
```

## ファイル命名規則

### shadcn/ui のスタイル（フラット）

shadcn/ui はフラットな命名規則（`button.tsx`, `dialog.tsx`）を採用している。

```
src/components/ui/
├── button.tsx
├── dialog.tsx
└── input.tsx
```

### 自作コンポーネントのスタイル（ディレクトリ方式）

自分で作るコンポーネントは `コンポーネント名/index.tsx` で統一する。

```
src/components/
├── ui/                    # shadcn → フラットのまま
│   ├── button.tsx
│   ├── dialog.tsx
│   └── input.tsx
├── pages/                 # ページ
│   └── Test/
│       └── index.tsx
└── SearchField/           # 自作コンポーネント → ディレクトリ方式
    ├── index.tsx
    ├── SearchField.test.tsx
    └── hooks.ts
```

`ui/` は shadcn の管轄、それ以外は自分のルール、と**レイヤーを分けて考える**のがよい。

## Storybook を導入する場合

Storybook を導入すると、Story ファイルがコンポーネントごとに増えるため、`ui/` もディレクトリ方式に寄せた方が管理しやすくなる。

```
src/components/ui/
├── Button/
│   ├── index.tsx
│   └── index.stories.tsx
├── Dialog/
│   ├── index.tsx
│   └── index.stories.tsx
└── Input/
    ├── index.tsx
    └── index.stories.tsx
```

### `shadcn add` との共存における注意点

ディレクトリ方式に変更した場合、`shadcn add` でコンポーネントを追加・更新すると `ui/button.tsx` のようにフラットに生成されるため、既存のディレクトリ構造と衝突する。

対処法：

1. **初回だけ `shadcn add` で生成し、手動でディレクトリに移す** — Story ファイルを追加するタイミングで移行する
2. **`shadcn add` は差分確認用に使い、手動でマージする** — 更新時にフラットに出力されたものを自分のディレクトリ構造に反映する

Storybook 導入の予定がなければ、今はフラットのままにしておいて、導入時にまとめて移行するのでも問題ない。
